# Button

<Tagline>A button indicates a possible user action.</Tagline>

<ComponentInfo
    usage={"import { Button, IconButton, ToggleButton, ToggleIconButton, ButtonGroup, ButtonAsLink } from \"@sharegate/orbit-ui\";"}
    ariaPath="#button"
    githubPath="/packages/components/src/button/src"
/>

## Usage

### Default

A default button.

<Preview>
    <Story name="default">
        <Button>Cutoff</Button>
    </Story>
</Preview>

### Icon

A button can contain [icons](?path=/docs/icon-gallery--page).

<Preview>
    <Story name="icon">
        <Button variant="outline" tone="basic">
            <LightbulbIcon />
            <Text>Cutoff</Text>
        </Button>
    </Story>
</Preview>

### End icon

*Non standard* end [icons](?path=/docs/icon-gallery--page) can be provided to handle special cases. However, think twice before adding *end* icons, *start* icons should be your go to.

<Preview>
    <Story name="end icon">
        <Button variant="outline" tone="basic">
            <Text>Cutoff</Text>
            <LightbulbIcon slot="end-icon" />
        </Button>
    </Story>
</Preview>

### Counter

A button can contain a [counter](?path=/docs/counter--default-story).

<Preview>
    <Story name="counter">
        <Button variant="outline" tone="basic">
            <Text>Cutoff</Text>
            <Counter variant="divider">60</Counter>
        </Button>
    </Story>
</Preview>

### Size

A button can vary in size.

<Preview>
    <Story name="size">
        <Inline alignY="center">
            <Button size="sm" variant="outline" tone="basic">Cutoff</Button>
            <Button variant="outline" tone="basic">Cutoff</Button>
        </Inline>
    </Story>
</Preview>

### Disabled

A button can be disabled.

<Preview>
    <Story name="disabled">
        <Button disabled>Cutoff</Button>
    </Story>
</Preview>

### Loading

A button can show a loading indicator.

<Preview>
    <Story name="loading">
        <Button loading variant="outline" tone="basic">Loading</Button>
    </Story>
</Preview>

### Fluid

A button can take the width of its container.

<Preview>
    <Story name="fluid">
        <Button fluid variant="outline" tone="basic">
            <LightbulbIcon />
            <Text>To infinity and beyond</Text>
        </Button>
    </Story>
</Preview>

### Tone

A button can use a different tones.

<Preview>
    <Story name="tone">
        <Inline>
            <Button tone="accent">Accent</Button>
            <Button tone="negative">Negative</Button>
            <Button variant="outline" tone="accent">Accent</Button>
            <Button variant="outline" tone="basic">Basic</Button>
            <Button variant="outline" tone="negative">Negative</Button>
        </Inline>
    </Story>
</Preview>

### Ghost

A button can be discrete.

<Preview>
    <Story name="ghost">
        <Inline>
            <Button tone="accent" variant="ghost">Accent</Button>
            <Button tone="basic" variant="ghost">Basic</Button>
            <Button tone="negative" variant="ghost">Delete</Button>
        </Inline>
    </Story>
</Preview>

### Link

A button can be rendered as a link by using the `ButtonAsLink` component.

<Preview>
    <Story name="as anchor">
        <ButtonAsLink href="https://sharegate.com" target="_blank" variant="outline" tone="basic">Cutoff</ButtonAsLink>
    </Story>
</Preview>

## Icon button

A button content can be a single icon. An accessible name must be provided through `aria-label` prop. See [WCAG practices](https://www.w3.org/TR/WCAG20-TECHS/ARIA6.html).

<Preview>
    <Story name="icon button">
        <Inline alignY="center">
            <IconButton size="sm" variant="outline" tone="basic" aria-label="Add"><AddIcon /></IconButton>
            <IconButton variant="outline" tone="basic" aria-label="Add"><AddIcon /></IconButton>
        </Inline>
    </Story>
</Preview>

## Button group

Buttons whose actions are related to each other can be grouped together.

<Preview>
    <Story name="button group">
        <ButtonGroup>
            <Button variant="outline" tone="basic">No, thanks</Button>
            <Button variant="outline" tone="basic">Remind me later</Button>
            <Button variant="outline" tone="basic">Rate Now</Button>
        </ButtonGroup>
    </Story>
</Preview>

### Orientation

A button group can render his items vertically.

<Preview>
    <Story name="button group vertical">
        <ButtonGroup orientation="vertical">
            <Button variant="outline" tone="basic">No, thanks</Button>
            <Button variant="outline" tone="basic">Remind me later</Button>
            <Button variant="outline" tone="basic">Rate Now</Button>
        </ButtonGroup>
    </Story>
</Preview>

### Alignment

A button group can change the alignment of his items.

<Preview>
    <Story name="button group alignment">
        <ButtonGroup align="end">
            <Button variant="outline" tone="basic">No, thanks</Button>
            <Button variant="outline" tone="basic">Remind me later</Button>
            <Button variant="outline" tone="basic">Rate Now</Button>
        </ButtonGroup>
    </Story>
</Preview>

## Toggle button

Offer a similar experience as a checkbox or radio with the appearance of a button.

<Preview>
    <Story name="toggle button">
        <Inline>
            <ToggleButton variant="outline" tone="basic" shape="circular">On</ToggleButton>
            <ToggleButton variant="outline" tone="basic">
                <LightbulbIcon />
                <Text>Cutoff</Text>
            </ToggleButton>
        </Inline>
    </Story>
</Preview>

### Toggle icon button

A toggle button content can be a single icon. An accessible name must be provided through `aria-label` prop. See [WCAG practices](https://www.w3.org/TR/WCAG20-TECHS/ARIA6.html).

<Preview>
    <Story name="toggle icon button">
        <ToggleIconButton variant="outline" tone="basic" aria-label="Activate">
            <CheckIcon />
        </ToggleIconButton>
    </Story>
</Preview>

### Toggle button group

Related toggle buttons can be group together in a [group](#button-group), a [checkbox group](?path=/docs/checkbox--default-story) or a [radio group](?path=/docs/radio-group--default-story):

<Preview>
    <Story name="toggle button group">
        <RadioGroup gap={2} orientation="horizontal">
            <ToggleButton value="none" variant="outline" tone="basic">None</ToggleButton>
            <ToggleButton value="1" variant="outline" tone="basic" shape="circular">1</ToggleButton>
            <ToggleButton value="2" variant="outline" tone="basic" shape="circular">2</ToggleButton>
            <ToggleButton value="3" variant="outline" tone="basic" shape="circular">3</ToggleButton>
            <ToggleButton value="4+" variant="outline" tone="basic" shape="circular">4+</ToggleButton>
        </RadioGroup>
    </Story>
</Preview>
