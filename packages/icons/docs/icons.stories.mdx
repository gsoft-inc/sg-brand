## Design specifications

All official SVG icons includes their padding and are generally available in two variants, 24px and 32px. If an icon is only available in 32px, he is considered universal.

## Icon dimensions

To define dimensions, a few options are available depending on whether you choose to include an icon as a [React component](#components) or as a [static file](#static-files).

### React components

If you choose to include an icon as a component, you don't have to worry about dimensions. Components accept a [`size`](#sizes) prop which will translate to the appropriate dimensions classes at render time.

Two types of components are available, [multi-variants](#multi-variants) components and [variants](#variants) components. They both share the same `size` prop but handles variants differently.

#### Multi-variants

Multi-variants components will decide for you which variant of an icon to render based on the `size` you specify. Use a multi-variants component if you don't have a strong opinion about how icons variants and icons dimensions should be paired. We recommend using multi-variants components whenever you can since this is the easiest way to ensure your icon is compliant with current and future versions of Orbit Design.

```jsx
import { CalendarMajorIcon } from "@sharegate/orbit-ui";

function render() {
    return <CalendarMajorIcon aria-label="Calendar" />;
}
```

#### Variants

Variants components let you decide at import time which variant of an icon to use. Use a variant component to pair an icon variant with any dimensions.

```jsx
import { CalendarMajorIcon24 } from "@sharegate/orbit-ui";

function render() {
    return <CalendarMajorIcon24 aria-label="Calendar" />;
}
```

#### Sizes

By default an icon component render as an `md` sized icon. To alter the dimensions of an icon, you can specify any of the following sizes: `"2xs"`, `"xs"`, `"sm"`, `"lg"`, `"xl"`, `"inherit"`.

> If your icon is intended to be used as content of an Orbit component, you won't have to specify the icon size since Orbit components already have pre-determined dimensions for their icons.

### Static files

Icons SVG files are available from the root of `@orbit-ui/icons` package. These files are usefull for non React apps or any use cases which cannot be done using a component. Whenever you can, you should prefer React components to static files.

Orbit Design provide icons with default dimensions, e.g. a `24px` variant will have a default viewport of 24x24 and a viewbox of `0 0 24 24`.

You can customize an icon dimensions with the `width` and `height` props of Orbit [styled-system](?path=/docs/style-props--page).

#### Image source

```jsx
import CalendarMajorIcon24 from "@orbit-ui/icons/icon-calendar-24.svg";

function render() {
    return <Img src={CalendarMajorIcon24} alt="Calendar" width={6} height={6} />;
}
```

#### CSS background

```css
.calendar-24 {
    background: url("~@orbit-ui/icons/icon-calendar-24.svg");
    width: var(--o-ui-sp-4);
    height: var(--o-ui-sp-4);
}
```

## Color

An SVG object get it's color from the [fill attribute](https://developer.mozilla.org/en-US/docs/Web/SVG/Tutorial/Fills_and_Strokes). When an SVG object doesn't define a `fill` attribute on his elements, CSS can be use to style the internal color of the elements of the objects with the `fill` property.

This means you can alter the color of an icon by specifying a new `fill` value. If the SVG already contains a `fill` attribute, please [fill an issue](https://github.com/gsoft-inc/sg-orbit/issues).

### React components

To set the color of an icon component, you can set the `fill` prop of Orbit [styled-system](?path=/docs/style-props--page).

```jsx
import { CalendarMajorIcon } from "@sharegate/orbit-ui";

function render() {
    return <CalendarMajorIcon aria-label="Calendar" fill="purple-8" />
}
```

### Image source

To set the color of an image, you can set the `fill` prop of Orbit [styled-system](?path=/docs/style-props--page).

```jsx
import CalendarMajorIcon24 from "@orbit-ui/icons/icon-calendar-24.svg";

function render() {
    return <Img src={CalendarMajorIcon24} alt="calendar" fill="purple-8" />;
}
```
