# Icons

<Tagline>Orbit Design provide a set of SVG icons built specifically for ShareGate products.</Tagline>

All official icons **includes their padding** and are available in `32px`.

Orbit UI make those icons available for products in two ways, *React components* and *static files*.

If you are looking for documentation about custom SVG icons (e.g. Azure specific icons), please refer to the [icon component](?path=/docs/icon--default-story) documentation.

## Dimensions

The dimensions of an icon should always match Orbit Design [spacing scale](?path=/docs/spacing--page#values) values. To define dimensions, a few options are available depending on whether you choose to include an icon as a [React component](#react-components) or as a [static file](#static-files).

### React components

If you choose to include an icon as a component, you don't have to worry about dimensions. Components accept a `size` prop that will translate to dimensions classes at render time.

Two types of components are available, _multi-variants_ components and _variants_ components. They both share the same `size` prop but handles variants differently.

#### Multi-variants

Multi-variants components will decide for you which variant of an icon to render based on the `size` you specify. Use a multi-variants component if you don't have a strong opinion about how icons variants and icons dimensions should be paired. We do recommend using multi-variants components whenever you can since this is the easiest way to ensure your icon is compliant with current and future versions of Orbit Design.

```jsx
import { CalendarIcon } from "@orbit-ui/react-components";

function render() {
    return <CalendarIcon aria-label="Calendar" />;
}
```

#### Variants

Variants components let you decide at import time which variant of an icon to use. Use a variant component, where available, to pair an icon variant with any dimensions.

```jsx
import { CalendarIcon24 } from "@orbit-ui/react-components";

function render() {
    return <CalendarIcon24 aria-label="Calendar" />;
}
```

#### Sizes

By default an icon component render as an `md` sized icon.

To alter the dimensions of an icon, you can specify any of the following sizes: `"2xs"`, `"xs"`, `"sm"`, `"lg"`, `"xl"`, `"inherit"`.

> If your icon is intended to be used as content of an Orbit UI component, you won't have to specify the icon size since Orbit UI components already have pre-determined dimensions for their icons.

### Static files

Icons SVG files are available from the root of `@orbit-ui/icons` package. These files are usefull for non React apps or any use cases which cannot be done using a component. Whenever you can, you should prefer React components to static files.

Orbit Design provide icons with default dimensions, e.g. a `24px` variant will have a default viewport of 24x24 and a viewbox of `0 0 24 24`.

You can customize an icon dimensions with either [Tachyons dimensions classes](?path=/docs/spacing--page#tachyons) (e.g. `w6 h6` for a `24px` variant icon or `w7 h7` for a `32px` variant icon) or CSS and Orbit UI [CSS spacing variables](?path=/docs/materials-spacing--page#values).

#### Image source

```jsx
import CalendarIcon32 from "@orbit-ui/icons/icon-calendar-32.svg";

function render() {
    return <img src={CalendarIcon32} alt="Calendar" className="w7 h7 stroke-marine-900" />;
}
```

#### CSS background

```css
.calendar-32 {
    background: url("~@orbit-ui/icons/icon-calendar-32.svg");
    width: var(--o-ui-global-scale-golf);
    height: var(--o-ui-global-scale-golf);
}
```

## Color

An SVG object get it's color from the [stroke attribute](https://developer.mozilla.org/en-US/docs/Web/SVG/Tutorial/Fills_and_Strokes). When an SVG object doesn't define a `stroke` attribute on his elements, CSS can be use to style the internal color of the elements of the objects with the `stroke` property.

This means that you can alter the color of an icon by specifying a new `stroke` value with either [Tachyons stroke classes](?path=/docs/colors--page#tachyons) or with CSS and [Orbit UI color variables](?path=/docs/foundation--page#colors).

<code>stroke</code> attribute. If the SVG contains a <code>stroke</code> attribute, please <ExternalLink href="https://github.com/gsoft-inc/sg-orbit/issues">fill an issue</ExternalLink>.

### React components

To set the color of an icon component, you can add [Tachyons stroke classes](?path=/docs/colors--page#tachyons) directly on the component with the `className` prop:

```jsx
import { CalendarIcon } from "@orbit-ui/react-components";

function render() {
    return <CalendarIcon className="stroke-marine-900" aria-label="Calendar" />
}
```

If your color is dynamic, you can use the `style` prop and [Orbit UI color variables](?path=/docs/foundation--page#colors):

```jsx
import { CalendarIcon } from "@orbit-ui/react-components";

function render() {
    return <CalendarIcon style={{ stroke: "var(--o-ui-global-marine-900)" }} aria-label="Calendar" />;
}
```

### Image source

To set the color of an image icon you can add [Tachyons stroke classes](?path=/docs/colors--page#tachyons) directly on the component with the `className` prop or use the `style` prop and [Orbit UI color variables](?path=/docs/colors--page#colors) to set a dynamic color:

```jsx
import CalendarIcon32 from "@orbit-ui/icons/icon-calendar-32.svg";

function render() {
    return <img src={CalendarIcon32} alt="calendar" className="stroke-marine-900" />;
}
```

### CSS background

You can set the color of a CSS background icon by setting an [Orbit UI color variables](?path=/docs/foundation--page#colors) to the `stroke` property of the class:

```css
.calendar-32 {
    background: url("~@orbit-ui/icons/icon-calendar-32.svg");
    stroke: var(--o-ui-global-marine-900);
}
```
