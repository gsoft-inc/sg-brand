# Message

<Tagline>A message is an element that displays a brief, important message in a way that attracts the user's attention without interrupting the user's task.</Tagline>

<ComponentInfo
    usage={"import { Message, Content } from \"@orbit-ui/react-components\";"}
    ariaPath="#alert"
    githubPath="/packages/react-components/src/message/src"
/>

## Usage

### Default

A default message.

<Preview>
    <Story name="default">
        <Message>Your email to <strong>booking@spacetravel.com</strong> has been sent!</Message>
    </Story>
</Preview>

### Icon

A message can have an [icon](?path=/docs/icon-gallery--page).

<Preview>
    <Story name="icon">
        <Message>
            <EmailIcon />
            <Content>Your email to <strong>booking@spacetravel.com</strong> has been sent!</Content>
        </Message>
    </Story>
</Preview>

### Rich content

A message can also accept rich content.

<Preview>
    <Story name="rich content">
        <Message>
            <EmailIcon />
            <Content>
                <Heading>booking@spacetravel.com</Heading>
                Your email has been sent! Tracking info available <TextLink href="#">here</TextLink>.
            </Content>
        </Message>
    </Story>
</Preview>

### Action

A message can accept an action [button](?path=/docs/button--default-story).

<Preview>
    <Story name="action">
        <Message>
            <EmailIcon />
            <Content>Your email to <strong>booking@spacetravel.com</strong> has been sent!</Content>
            <Button>Undo</Button>
        </Message>
    </Story>
</Preview>

### Dismissible

A message can be dismissible by specifying an `onDismiss` handler.

<Preview>
    <Story name="dismissible">
        <Message onDismiss={() => {}}>
            <EmailIcon />
            <Content>Your email to <strong>booking@spacetravel.com</strong> has been sent!</Content>
            <Button>Undo</Button>
        </Message>
    </Story>
</Preview>

### Informative

Use an informative message to notify a user of a change that's relevant for them to achieve their current goal.

<Preview>
    <Story name="informative">
        <Message variant="informative" onDismiss={() => {}}>
            <Content>Your email to <strong>booking@spacetravel.com</strong> has been sent!</Content>
            <Button>Undo</Button>
        </Message>
    </Story>
</Preview>

### Warning

Use a warning message to help a user avoid an issue.

<Preview>
    <Story name="warning">
        <Message variant="warning" onDismiss={() => {}}>
            <Content>We are having trouble contacting the email server.</Content>
            <Button>Retry</Button>
        </Message>
    </Story>
</Preview>

### Positive

Use a positive message to inform a user that an action has completed successfuly.

<Preview>
    <Story name="positive">
        <Message variant="positive" onDismiss={() => {}}>
            <Content>Your email to <strong>booking@spacetravel.com</strong> has been sent!</Content>
            <Button>Undo</Button>
        </Message>
    </Story>
</Preview>

### Negative

Use a negative message to help a user understand a critical issue at the system level.

<Preview>
    <Story name="negative">
        <Message variant="negative" onDismiss={() => {}}>
            <Content>Cannot contact email server.</Content>
            <Button>Retry</Button>
        </Message>
    </Story>
</Preview>

A message can handle `show` state in controlled mode.
